# 가계부 - 예산 점검 페이지

월별 예산 계획(지출예정)과 실제 지출(실제지출)을 비교하고, 정산에서 잔여 현금을 확인하는 개인용 원페이지 앱입니다.

---

## 제작 목적

- **개인 월별 예산 관리**: 계획한 지출(지출예정)과 실제 지출을 한 화면에서 기록·비교하여, 월 단위로 잔여 현금을 파악한다.
- **재사용 가능한 기본 설정**: 목적별 통장/카드, 분류별 생활비, 결제 수단을 한 번 설정해 두고 여러 달에 걸쳐 재사용한다.
- **단일 사용자·원페이지**: 로그인 없이 접속 코드로 진입하며, 한 페이지에서 기본 설정·월별 데이터·정산까지 모두 처리한다.

---

## 설계 구조

```
[브라우저]
    │
    ▼
Next.js App (app/page.tsx)  ← 단일 페이지, 클라이언트 상태 + API 호출
    │
    ├── /api/bootstrap        → 초기 로드 (월 목록, 고정 데이터, 선택 월 데이터)
    ├── /api/fixed/purpose    → 목적별 통장·카드 분류 (PUT)
    ├── /api/fixed/living     → 분류별 생활비 (PUT)
    ├── /api/fixed/pay-categories → 결제 수단 (PUT)
    ├── /api/months           → 월 생성 (POST)
    └── /api/months/[yyyymm]  → 월별 데이터 조회·수정·삭제 (GET/PUT/DELETE)
            │
            ▼
    Supabase (Postgres)  ← RLS로 owner_id 단일 사용자 격리
```

- **프론트**: Next.js App Router, `app/page.tsx` 단일 페이지에서 모든 섹션 렌더. 편집/보기 모드·정렬·접속 코드( AuthGate )는 클라이언트 상태로 처리.
- **API**: Route Handler로 고정 데이터(purpose, living, pay-categories)와 월별 데이터(months, planned_items, actual_items) CRUD. 서버에서만 Supabase Service Role 사용.
- **정산 계산**: 클라이언트 `lib/calc/settlement.ts`에서 월급·이월·지출예정·실제지출을 합산해 계획/실제 순현금·카테고리 차이·카드별 합계 산출.

## 각 영역 설명

| 영역 | 설명 |
|------|------|
| **접속 코드** | 최초 진입 시 `NEXT_PUBLIC_ACCESS_CODE`와 일치하는 코드 입력 시에만 메인 페이지 접근. 세션 동안만 유지. |
| **월 선택·생성** | 현재 조회 중인 월을 표시하고, 이전/다음 달로 이동. «이전 달 생성» «다음 달 생성»으로 빈 월 레코드를 추가. |
| **목적별 통장 & 카드 사용 분류** | 생활비/구독·경조사 등 목적별로 통장·연결 카드·월 한도·사용 요약·유의사항을 등록. 정산 화면이 아닌 기본 틀 설정용. |
| **분류별 생활비** | 연간·월간 구독, 기본생활, 낭비 등 분류별로 항목 이름·기준 금액·결제 수단·메모 관리. 역시 기본 설정이며, 분류/금액 기준 테이블 정렬·분류별 셀 색상 적용. |
| **결제 수단 설정** | 카드/계좌 등 결제 수단 라벨을 등록. 지출예정/실제지출의 «카드» 선택 옵션 및 정산의 «결제수단별 실제지출 합계»에 사용. |
| **지출예정 / 실제지출** | **월급·추가 수입(이월 등)** 을 입력하고, **지출예정** 테이블(분류·항목·금액·카드·메모)과 **실제지출** 테이블을 각각 수정. 지출예정을 실제지출로 복사 가능. 분류별 지출 통계(예정/실제) 표시. |
| **정산** | 이달 총 수입(월급+추가수입), 계획/실제 총지출·순현금, 순현금 차이를 카드로 표시. 카테고리별 «계획 − 실제» 차이 테이블, 결제수단별 실제지출 합계 표시. |
| **하단 액션** | «삭제»(현재 월 데이터 삭제), «저장»(현재 월 월급·추가수입·지출예정·실제지출 일괄 저장). «계획 저장»은 고정 데이터(목적별·분류별) 저장. |
| **데이터 가져오기** | 다른 월의 데이터를 선택해 현재 월로 복사(월급·이월·지출예정·실제지출 등). |

## 주요 기능 및 사용 방법

### 1. 접속 코드 (선택)

- **설정**: 환경 변수 `NEXT_PUBLIC_ACCESS_CODE`에 원하는 코드를 넣으면 활성화됩니다.
- **사용**: 앱 접속 시 코드 입력 화면이 나옵니다. 올바른 코드 입력 후 «확인»을 누르면 메인 페이지로 이동합니다. 브라우저 탭을 닫으면 다시 코드 입력이 필요합니다.

---

### 2. 기본 설정 (목적별 통장·분류별 생활비·결제 수단)

- **목적별 통장 & 카드 사용 분류**
  - «수정» 클릭 → 목적(생활비/구독·정기결제/경조사비), 통장명, 연결 카드, 금액(월 한도), 사용 요약, 유의사항 입력 → «행 추가»로 항목 추가 가능 → «수정 완료» 후 **계획 저장** 버튼으로 반영. «취소» 시 저장 없이 편집 종료.
- **분류별 생활비**
  - «수정» 클릭 → 분류(연간·월간 구독, 기본생활, 낭비 등), 이름, 기준 금액, 결제 수단, 메모 입력 → «수정 완료» 후 **계획 저장**. «취소» 시 변경 사항 폐기.
- **결제 수단 설정**
  - «결제 수단 설정» 버튼 클릭 → 모달에서 카드/계좌 이름 입력 후 «행 추가»·삭제 → «저장»으로 반영. 지출예정/실제지출의 «카드» 선택 목록과 정산의 «결제수단별 실제지출 합계»에 사용됩니다.
- **참고**: 다른 영역을 수정 중일 때 «수정»을 누르면 “다른 영역 수정 저장 후 전환할까요?” 알림이 뜹니다. 확인 시 해당 영역을 저장한 뒤 새 영역 편집 모드로 들어갑니다.

---

### 3. 월 선택·이전/다음 달 생성

- **월 이동**: 상단의 «이전 달» «다음 달» 화살표 또는 월 제목 클릭으로 월 선택기를 열어 다른 월로 이동합니다.
- **다음 달 생성**: «다음 달 생성» 클릭 → 확인 시 새 월(yyyy-mm)이 생성되고 해당 월로 이동. 월급·이월·지출예정 등은 비어 있으므로 직접 입력하거나 «데이터 가져오기»로 다른 월에서 복사할 수 있습니다.
- **이전 달 생성**: «이전 달 생성» 클릭 → 확인 시 현재 월보다 이전 월이 없으면 그 달이 생성됩니다.

---

### 4. 월간 운영 (지출예정 / 실제지출)

- **월급·추가 수입**
  - 상단 «월급» 칸 클릭 시 숫자 표시/숨김 토글. «추가 수입»은 유형(이월금액/인센티브 등) 선택 후 금액 입력. **지출예정** 또는 **실제지출** 중 하나라도 «수정» 모드일 때만 편집 가능.
- **지출예정**
  - «지출예정» 영역의 «수정» 클릭 → 분류·항목·금액·카드·메모 입력, «행 추가»로 항목 추가, 행별 «삭제» 가능 → «수정 완료» 후 하단 **저장** 버튼으로 반영. «취소» 시 서버에서 다시 불러와 변경 사항이 사라집니다.
- **실제지출**
  - «실제지출» 영역의 «수정» 클릭 → 위와 동일하게 입력·추가·삭제. «예정 지출을 실제 지출로 복사» 버튼으로 지출예정 항목을 그대로 실제지출로 가져올 수 있습니다. «수정 완료» 후 **저장**으로 반영, «취소» 시 변경 사항 폐기.
- **저장**: 화면 하단 **저장** 버튼은 **현재 선택된 월**의 월급·추가수입·지출예정·실제지출을 한 번에 저장합니다. 수정 후 반드시 저장해야 서버에 남습니다.

---

### 5. 정산 보기

- **카드**: 이달 총 수입(월급+추가수입), 계획 총지출, 실제 총지출, 계획/실제 순현금, 순현금 차이가 자동 계산되어 표시됩니다.
- **카테고리별**: 고정저축·유동지출 등 카테고리별로 «계획 − 실제» 차이가 표시됩니다. 양수면 계획보다 덜 쓴 것, 음수면 더 쓴 것입니다.
- **결제수단별 실제지출 합계**: 결제 수단별로 실제지출 합계가 표시됩니다. «결제 수단 설정»에서 등록한 수단만 나옵니다.

---

### 6. 데이터 가져오기

- **방법**: 우측 상단 «데이터 가져오기» 클릭 → 모달에서 **가져올 월** 선택 → 확인 시 해당 월의 월급·추가수입·지출예정·실제지출이 **현재 선택된 월**로 복사됩니다. 기존 현재 월 데이터는 덮어씌워지므로 필요한 경우 미리 백업하세요.

---

### 7. 삭제·계획 저장

- **삭제**: 하단 «삭제»는 **현재 선택된 월**의 월 데이터(월급·이월·지출예정·실제지출)만 삭제합니다. 월이 하나뿐이면 삭제가 막힐 수 있습니다.
- **계획 저장**: «계획 저장»은 **목적별 통장·분류별 생활비** 등 고정 설정만 저장합니다. 지출예정/실제지출을 수정 중이면 “다른 영역 수정 완료 후 저장할까요?” 알림이 뜹니다.

## 기술 스택

- Next.js (App Router)
- Supabase (Postgres, RLS)
- Tailwind CSS

## 환경 변수

`.env.local` 파일을 만들고 아래 값을 설정하세요.

```bash
cp .env.example .env.local
```

- `SUPABASE_URL`: Supabase 프로젝트 URL
- `SUPABASE_SERVICE_ROLE_KEY`: 서버 전용 키
- `NEXT_PUBLIC_SUPABASE_URL`: 클라이언트 URL (선택)
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`: 클라이언트 anon 키 (선택)
- `APP_OWNER_ID`: 단일 사용자 식별자
- `NEXT_PUBLIC_ACCESS_CODE`: (선택) 메인 페이지 접근용 코드. 설정 시 진입 시 코드 입력 필요

## 로컬 실행

```bash
npm install
npm run dev
```

브라우저에서 `http://localhost:3000` 접속.

## Supabase 마이그레이션

`supabase/migrations/001_init.sql` 파일을 SQL Editor에서 실행하세요.

생성 대상:
- `profiles`
- `purpose_accounts`
- `living_groups`
- `months`
- `planned_items`
- `actual_items`
- `monthly_summary` view

## 배포 (Vercel + 도메인)

1. GitHub에 레포 푸시
2. Vercel에서 프로젝트 Import
3. Vercel 환경 변수에 `.env.local` 동일 값 등록
4. 도메인 연결 후 DNS 설정
5. 배포 완료 후 앱 경로를 개인용 비공개 URL로 사용

## 백업/복구 운영 루틴

- 월 1회 백업:
  - Supabase Table Editor에서 CSV Export (`months`, `planned_items`, `actual_items`)
  - 필요 시 프로젝트 백업(스냅샷)
- 복구:
  - CSV Import 또는 SQL Insert로 해당 월 데이터 복원

## 참고

- 본 앱은 로그인 UI가 없으므로 공개 URL 관리가 중요합니다.
- 서비스 롤 키는 서버에서만 사용해야 하며 절대 클라이언트 코드에 노출하면 안 됩니다.
